# Security Policy

## Supported Version

Security fixes are handled for the latest source release on the `main` branch.

## Reporting Bugs Or Security Issues

If you find a bug, crash, unsafe behavior, or possible security issue, please open a GitHub issue and include enough detail for the problem to be reproduced and fixed.

Please include:

- A short description of what happened.
- Steps to reproduce the issue.
- What you expected to happen.
- What actually happened.
- Your Windows version.
- Your Node.js and Python versions, if relevant.
- The TTS engine and voice you were using.
- Any relevant screenshots, screen recordings, logs, or error messages.

Please redact secrets before uploading anything. Do not include Twitch client secrets, access tokens, `.env` contents, private paths you do not want public, or other credentials.

If the issue can be triggered by a chat message, include the exact message text when it is safe to share. If the message contains private or harmful content, replace sensitive parts with placeholders while keeping the structure of the message intact.

## Response

Reports will be reviewed as time allows. Reproducible reports with clear steps and evidence are the fastest to fix.
